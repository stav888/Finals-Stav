Question (from finals page — Q9):
What is the difference between a Docker image and a Docker container? Use an analogy to explain. Common analogies: class vs instance, recipe vs cake, blueprint vs building. Any analogy that works is fine.

Expected: short analogy and clear mapping between image vs container.

Answer:

A Docker image is a read-only snapshot that describes how to build a runnable environment (it contains the filesystem, dependencies, and metadata). A Docker container is a running instance created from that image — it’s the active process with its own writable layer and runtime state.

Analogy:
Think of a Docker image as a recipe or blueprint and a Docker container as the finished cake or building. The recipe (image) lists ingredients and steps; the cake (container) is what you get when you execute the recipe — you can eat it, run it, and it has state (slices removed, frosting added) that the original recipe file does not.

One-sentence summary:
Image = immutable template/blueprint; container = running instance/process derived from that template with runtime state.
