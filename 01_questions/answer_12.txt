Question (from finals page — Q12):
What are Agent Skills in the context of AI coding assistants? How are they different from just writing instructions in a plain prompt? Show a minimal example of what a skill's `.md` metadata block might look like. Think about: a skill has a name, a description, and a path to a detailed file.

Expected: short definition of Agent Skills, contrast with plain prompts, and a minimal `.md` metadata example.

Answer:

Agent Skills are small, discoverable metadata + implementation packages that expose domain-specific capabilities to AI assistants in a structured way. Unlike informal instructions embedded in a prompt, a Skill includes a machine-readable metadata block (name, description, trigger conditions, and file location) plus the detailed instructions or code in a separate file. This makes Skills discoverable, safely activatable, and easier to maintain and reuse across assistants.

Why different from prompt-only instructions:
- Discoverable: Skills are listed and can be auto-selected by the assistant when relevant.
- Safer and versioned: stored in files with access controls rather than ad-hoc prompt text.
- Reusable: the same skill can be used by multiple assistants or workflows.

Minimal skill metadata example:

<skill>
  <name>mongodb-query-optimizer</name>
  <description>Help optimize MongoDB queries and suggest indexes when queries are slow.</description>
  <file>skills/mongodb-query-optimizer/SKILL.md</file>
</skill>

One-sentence summary:
Agent Skills package domain knowledge and actions as structured, discoverable files (metadata + implementation), unlike ephemeral prompt text.
