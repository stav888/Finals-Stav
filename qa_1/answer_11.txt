Question (from finals page — Q11):
What is MCP (Model Context Protocol)? What problem does it solve for AI coding assistants like GitHub Copilot? Name two examples of things an MCP server might expose to an AI assistant. Think about: how does a language model normally know about your database, your files, or your GitHub issues? What does MCP standardize?

Expected: short definition of MCP, the problem it solves (standardized access to external context/tools), and two example MCP server capabilities.

Answer:

MCP (Model Context Protocol) is a standard that defines how external context providers (MCP servers) expose structured data and tools to language models and AI assistants. It solves the problem of ad-hoc, brittle integrations by providing a consistent interface for assistants to request and receive contextual information (files, database rows, search results, tool invocations) and to perform actions safely.

Two examples an MCP server might expose:
- Filesystem access: list files, read file contents, watch directories — so the assistant can inspect the user's repo.
- GitHub API proxy: query issues, pull requests, or commit history — so the assistant can reference or create GitHub resources without embedding tokens in prompts.

One-sentence summary:
MCP standardizes how assistants access external context and tools, making integrations safe, discoverable, and consistent across environments.
